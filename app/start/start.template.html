Directives share controller:s, parent controller

If this controller was named ctrl, it would have shared scope with sub directives (which have scope:false)

{{directiveShareParentCtrl.person.food}}

Directives share a controller

Directive a, b, c must share scope (scope:false) for access to same controller in bindings
Directives also gets access to directiveShareParentCtrl:s properties and can bind to them

Directives also share directiveShareGameFactory, they can share it and work with bindings, even with isolate scope (scope:{})

Sub Controller Bindings

If controller as properties name are different (parents and child), then the child can bind to the parents properties.
If controller as properties name the same, then child can not bind to parents properties.

someControllerParent name: {{ someCtrlA.name }}
someControllerChild name: {{ someCtrlB.name }}
someControllerParent name: {{ someCtrlA.name }}
